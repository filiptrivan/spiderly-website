title: Attributes

Spiderly uses C# attributes to configure entity behavior, relationships, UI generation, and more. Attributes are grouped into the following categories:

## [Relationships](/docs/attributes/relationships)

Attributes for defining entity relationships — many-to-one, many-to-many, cascade behavior, and display names.

**Attributes:** M2M, M2MWithMany, WithMany, CascadeDelete, SetNull, DisplayName, ComplexManyToManyList, SimpleManyToManyTableLazyLoad

## [UI](/docs/attributes/ui)

Attributes that control how entities and properties are rendered in the Angular admin panel — control types, layout, ordering, and permissions.

**Attributes:** UIControlType, UIControlWidth, UIDoNotGenerate, UIPanel, UIPropertyBlockOrder, UIOrderedOneToMany, UITableColumn, UIAdditionalPermissionCodeForInsert, UIAdditionalPermissionCodeForUpdate

## [Code Generation](/docs/attributes/code-generation)

Attributes that control what gets generated — DTOs, services, mappers, and source generator output.

**Attributes:** BlobName, IncludeInDTO, ExcludeFromDTO, ExcludeServiceMethodsFromGeneration, GenerateCommaSeparatedDisplayName, ProjectToDTO, CustomMapper, Output

## [General](/docs/attributes/general)

Attributes for authentication, authorization, controller routing, and UI behavior.

**Attributes:** AuthGuard, DoNotAuthorize, Controller, SkipSpinner
